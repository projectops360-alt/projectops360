# specs/

Detailed feature & technical specifications. Create one file per feature **when it is being
designed or significantly changed** — before writing code (AI rule #2/#11).

**Naming:** `SPEC-<area>-<short-name>.md` (e.g. `SPEC-status-engine-living-graph-wiring.md`).

**Each spec should cover:** problem, expected behavior, affected Capability/Feature IDs,
data/schema, APIs/surfaces, dependencies, regression risks, test plan, related ADRs, rollout.

Empty for now — specs are added as work is planned.
