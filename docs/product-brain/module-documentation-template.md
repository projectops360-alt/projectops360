# Module Documentation Template

> Copy this template to document any ProjectOps360° module inside Product Intelligence. The
> [Living Graph doc](12-living-graph-strategy.md) is the worked example. Keep it honest: mark
> **Unknown / Documentation pending review** rather than overstating. Update the
> [Module Catalog](22-modules.md) and the [Capability Registry](05-capability-registry.md) when
> you add or change a module doc ([Governance Rules](23-governance-rules.md) → *Product
> Intelligence First*).

---

# <Module Name>

## 1. Module Purpose
What does this module do? (Plain executive language.)

## 2. Product Role
Why does this module exist in ProjectOps360°? Which [Strategic Pillar](02-strategic-pillars.md)?

## 3. Primary Users
Who uses it? (PMO · PM · Executives · Project team · Admins · Isabella/AI.)

## 4. Core Workflows
What can users actually do here?

## 5. Data Model
What data does it read and write? (Tables, key entities.)

## 6. AI Capabilities
What AI does it use? What may the AI do — and explicitly NOT do?

## 7. Deterministic Logic
What non-AI engines/calculations does it depend on?

## 8. Permissions
Who can see / create / update / delete / approve / export? (Server-enforced.)

## 9. Connected Modules
What does it depend on or feed?

## 10. Current Implementation
What exists today? Audited % and implemented parts (classify each: Implemented · Partial ·
Missing · Broken · Ad-hoc · Needs refactor · Should not be touched now).

## 11. Known Gaps
What is missing? For each: current → expected · risk · path · priority · needs-ADR? · needs-PI-update?

## 12. Risks / Anti-patterns
What should never happen? What change would be rejected?

## 13. Future Roadmap
What comes later, in sequence?

## 14. Related ADRs
Which [ADRs](07-adr-index.md) govern this module?

## 15. Capability IDs
Which [capabilities](05-capability-registry.md) (CAP-xxx) does it own/support?

## 16. Last Reviewed
Date · reviewer · "documentation only / runtime changed?".
