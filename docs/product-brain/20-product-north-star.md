# 20 — Product North Star

How a user should **feel** when using ProjectOps360°. This is the experiential target every
feature serves. It is about experience, not implementation. When a design decision is
ambiguous, choose the option that makes more of these statements true.

---

## The feelings we are designing for

- **"I always know what to do next."** — The system surfaces the next best action, not a wall
  of tasks.
- **"I never feel lost."** — There is always a way back to understanding: the graph, Isabella,
  the Command Center.
- **"I trust every recommendation."** — Recommendations are explainable, evidence-backed, and
  honest about uncertainty.
- **"The system remembers for me."** — Context, decisions, and history are never lost; I don't
  have to re-explain my project.
- **"I understand project impact instantly."** — I can see how a delay, a blocker, or a capacity
  shortfall ripples through schedule, budget, and the critical path.
- **"The project explains itself."** — I can ask "why is this like this?" about anything and get
  a clear, true answer.
- **"I feel in control, not buried."** — Administration recedes; understanding and decision-making
  come forward.
- **"It feels like it's thinking with me."** — The AI workforce is a partner, not a form.

## Anti-feelings (failure states to avoid)

- "I have to keep everything in my head."
- "I don't trust what the dashboard says."
- "I can't tell if this is actually blocked or just waiting."
- "I did this work before and it disappeared."
- "It's just another tool to update."

---

> If a feature does not move a user from an anti-feeling toward a North Star feeling, question
> whether it should be built.
