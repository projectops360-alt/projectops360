## EN: How to read the Process Intelligence Canvas

The canvas is an analytical SVG map derived exclusively from Project Event Graph business events. Nodes are activities (registry event types) showing frequency, case count and average incoming waiting; edges connect consecutive activities, with thickness proportional to frequency. The dominant path — the most frequent variant's exact sequence — is drawn solid in the brand color; secondary flows are lighter; rework transitions (the target activity had already happened in the same case) are dashed red AND always carry a text marker "↩ rework", never color alone. A bottleneck badge appears when the calculated score reaches 0.7 — the score is normalized average incoming waiting × frequency, computed by the projection engine, never decorated by hand. Users can filter by minimum frequency (the level-of-detail control that keeps 200-node graphs usable), isolate a single variant, show only rework plus the dominant path, zoom, and drill down: at organization level each case is a project journey; selecting a project switches cases to business-object journeys inside it. Clicking any node or edge opens the evidence drawer with its metrics, the projection's data-quality score, events used versus seen, and the standing limitations (temporal order is not causality; the only source is project_event_log). Saved views persist filter preferences per organization and level in the browser only — presentation, never business data.

## ES: Cómo leer el Canvas de Process Intelligence

El canvas es un mapa SVG analítico derivado exclusivamente de eventos de negocio del Project Event Graph. Los nodos son actividades (tipos de evento del registro) con frecuencia, número de casos y espera media de entrada; las aristas conectan actividades consecutivas, con grosor proporcional a la frecuencia. La ruta dominante — la secuencia exacta de la variante más frecuente — se dibuja sólida en el color de marca; los flujos secundarios son más tenues; las transiciones de retrabajo (la actividad destino ya había ocurrido en el mismo caso) son rojas discontinuas Y siempre llevan el marcador de texto "↩ retrabajo", nunca solo color. La insignia de cuello de botella aparece cuando el puntaje calculado llega a 0.7 — el puntaje es la espera media de entrada normalizada × frecuencia, computado por el motor de proyección, nunca decorado a mano. El usuario puede filtrar por frecuencia mínima (el control de nivel de detalle que mantiene usables grafos de 200 nodos), aislar una variante, mostrar solo retrabajo más la ruta dominante, hacer zoom y profundizar: a nivel organización cada caso es el recorrido de un proyecto; al seleccionar un proyecto los casos pasan a ser recorridos de objetos de negocio dentro de él. Al hacer clic en cualquier nodo o arista se abre el drawer de evidencia con sus métricas, el puntaje de calidad de datos de la proyección, eventos usados frente a vistos y las limitaciones vigentes (el orden temporal no es causalidad; la única fuente es project_event_log). Las vistas guardadas persisten preferencias de filtros por organización y nivel solo en el navegador — presentación, nunca datos de negocio.
